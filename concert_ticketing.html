동시성 트래픽을 고려한 콘서트 좌석 예약 시스템 구축 (항해99 팀 프로젝트)

📘 프로젝트 개요 (2024)

  • 대기열 기반 콘서트 좌석 예매 시스템 설계 및 구현
  • 실시간 좌석 조회, 임시 배정, 결제 기능 포함
  • 멀티 인스턴스 환경에서도 동시성 이슈 없이 안정 작동

🛠 주요 기능 (핵심 API 흐름)

  • 유저 토큰 발급: 대기열 참여자에게 UUID 토큰 발급
  • 좌석 조회: 예약 가능한 날짜 및 좌석 정보 제공
  • 좌석 예약: 5분간 임시 배정 후 미결제 시 자동 해제
  • 잔액 충전/조회: 예매 시 잔액 확인 및 제한 처리
  • 결제 처리: 완료 시 소유권 부여 및 토큰 만료 처리

🏗 모듈 설계 및 구조 (도메인 기반)

  • ConcertModule: 콘서트 정보, 좌석 조회
  • ReservationModule: 예약 로직 및 임시 배정 처리
  • PaymentModule: 결제 처리 및 소유권 이전
  • UserModule: 유저 잔액 관리 포함 (BalanceModule)
  • QueueModule: 대기열 토큰 검증 및 유효성 체크
  • → 클린 아키텍처 기반, 유즈케이스 계층 + REST 분리

🔄 기술적 도전 및 해결 (성능 & 동시성)

  • Redis 기반 분산 락으로 좌석 동시 예약 문제 해결
  • 임시 예약 & 결제 상태 분리로 유연한 트랜잭션 처리
  • 사용자 UUID 기반 대기열 접근 제어 로직
  • 상태 저장 Redis + 무상태 API 구조로 다중 인스턴스 대응

🧪 테스트 및 안정성 (검증 & 테스트)

  • 단위 테스트 작성으로 각 기능 검증
  • 멀티 인스턴스 환경에서 동일 유저 동시 접근 충돌 없음 확인

⚙ 기술 스택 (Backend & Infra)

  • NestJS, Redis, MySQL, Prisma
  • Docker 기반 컨테이너 환경
  • TypeScript, Swagger, JWT 인증

  • ┣ 📂domain
  • ┃ ┣ 📂model – 도메인 처리 모델델
  • ┃ ┣ 📂service – 도메인 서비스
  • ┣ 📂application
  • ┃ ┣ 📂usecases – 유즈케이스
  • ┃ ┗ 📂command – 명령 모델
  • ┣ 📂infra
  • ┃ ┣ 📂entity – DB테이블 모델
